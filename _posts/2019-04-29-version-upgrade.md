前端框架看久了，总会产生厌倦感。

很奇怪，作为一个后端程序猿不是应该喜欢那种土里土气、简单上手的框架么。

## React

大概去年参加 React 项目的时候，就想用新技术好好翻新一下。

还正儿八经的设计数据库表结构，写写画画准备来个动态工程。

可能战线拉的太长了，就无疾而终了。

于是乎，砍掉很多功能，并解决了很多坑一样的玩意之后，终于从零搭建了起来。

## Markdown

轻量级、方便随意的书写语法。

当然，正规的文档还是的靠 Office Word。

## React-markdown

经过大量的试错，最终实现了 Markdown 文档的解析。

从此，集美貌、才华于一体。

## Material-ui

等做完简单的移动适配之后，傻叉的发现为什么不用成熟的前端框架来实现？

现阶段很多的开源前端框架都默认实现了，电脑、移动端页面适配功能。做的很精美还没有 BUG。

## 思源宋体

Adobe与Google所领导开发的开放源代码字体家族 —— 饮水思源。

开放源代码字体，是开源、免费的意思吧。

## 删减功能

新版本去掉了很多花里胡哨的功能。

模糊查询、RSS、标签、disqus 之类的。

毕竟还是自娱自乐型的网络博客，还是简洁一些，关注在内容上吧。

## 旧文

其实，是想要删掉的。

现在回首，以前都写的什么玩意。但是又想想，勉强算是对未来幼稚的挣扎。

就保留在 Archives 吧。

## Nginx

本地运行没问题的情况下，打包部署出现了各种404。

无奈，本地安装 Nginx 各种调试。囧。

## 预期

希望独立设计更好的前端框架。
