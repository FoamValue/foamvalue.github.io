Java 企业级部署一般都选用 Linux 服务器，相对于微软的 Windows Server 服务器来说，更加的稳定可靠。但在配置和常规的维护上，却要比 Windows Server 服务器更加困难，也许但看到那单调的黑色背景和一串串的字母，就能催眠你了吧~~  
首先，超级简单的介绍下 CentOS ，这是众多 Linux 系统中之一，直接来源于 Red Hat Enterprise Linux（著名的商业发行版）。为了各种省钱的理由，一般 Java 企业级应用都会选择 CentOS 作为服务器系统，经济且实惠，哦耶~  
系统安装，请直接下载镜像文件（ISO），制作 U 盘工具后自行安装（和 Windows 一样简单）。

## 系统信息
查看系统版本：lsb_release \-a  
查看硬盘容量：df \-h  
查看内存容量：grep MemTotal /proc/meminfo

## 普通用户
创建普通用户，用来部署和简单的维护工作。  
创建用户：useradd foamvalue  
设置密码：passwd foamvalue  
这样创建，是不能使用 sudo 命令的，还需要配置 /usr/sbin/visudo 文件，在“root   ALL=(ALL)    ALL”下，新增一行“foamvalue   ALL=(ALL)   ALL”。由于 /usr/sbin/visudo 是二进制文件，所以在 sudo vi /usr/sbin/visudo 打开乱码的情况下，直接 sudo /usr/sbin/visudo 会有意想不到的惊喜哟~（vi 命令使用 sudo yum install vim，使用方式请 google~）

## 安装 jdk
本文使用的是 jdk\-6u45\-linux\-x64\-rpm.bin 进行安装的，如果你下载的是免安装的 tar.gz 压缩包的话。在解压缩后，请自行跳跃到 jdk 环境变量的配置上。  
授予可执行权限：chmod 755 jdk\-6u45\-linux\-x64\-rpm.bin  
执行安装程序：sudo ./jdk\-6u45\-linux\-x64\-rpm.bin  
查看 java 版本：java \-version  
此时，系统仍是 OpenJDK 版本。所以，需要手动删除 OpenJDK 和配置 jdk 环境变量。  
查看安装的 Java：rpm \-qa | grep java  
删除: sudo rpm \-e \-\-nodeps tzdata\-java\-2012c\-1.el6.noarch  
删除: sudo rpm \-e \-\-nodeps java\-1.6.0\-openjdk\-1.6.0.0\-1.45.1.11.1.el6.x86_64  
查看安装的 gcj：rpm \-qa | grep gcj  
删除: rpm \-e \-\-nodeps java\-1.5.0\-gcj\-1.5.0.0\-29.1.el6.x86_64  
配置环境变量：vi /etc/profile  
在文件最后添加：  
\#set java environment  
export JAVA_HOME=/usr/java/jdk1.6.0_45/  
export JRE_HOME=${JAVA_HOME}/jre  
export CLASSPATH=.:${JAVA_HOME}/lib:${JRE_HOME}/lib  
export PATH=${JAVA_HOME}/bin:${JRE_HOME}/bin:$PATH  
**Java 目录在/usr/java/jdk1.6.0_45/**  
使配置生效：source /etc/profile  
此时，在执行 java \-versin 可以发现已经正确配置好了。

## 结束语
以上，一个基础的环境就搭建好了。
